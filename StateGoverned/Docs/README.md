# Documentación del Proyecto StateGoverned

Esta carpeta contiene la documentación conceptual del proyecto `StateGoverned`.  
No describe el código línea por línea: explica las **ideas, decisiones y principios** que gobiernan su diseño.

Funciona como material de referencia para entender el proyecto como **caso de estudio arquitectónico**.

## Contenido

Los documentos clave que estructuran el proyecto son:

- [**ARCHITECTURE.md**](ARCHITECTURE.md)  
  Describe las decisiones arquitectónicas fundamentales del sistema y los principios que gobiernan su diseño.  
  Explica **por qué el sistema está construido de esta manera** y qué trade-offs se asumieron.

- [**TDD-AND-STATE-DRIVEN-DESIGN.md**](TDD-AND-STATE-DRIVEN-DESIGN.md)  
  Explica el rol del Desarrollo Guiado por Tests (TDD) como herramienta de diseño, y su integración con una arquitectura guiada por estado.  
  Muestra cómo los tests influyen directamente en la forma del dominio y sus APIs.
